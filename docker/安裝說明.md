# 建安 AI 系統安裝說明

> 適用版本：Windows 10 / 11

---

## 系統需求

在安裝之前，請確認您的電腦符合以下最低規格：

| 項目 | 最低需求 |
|------|---------|
| 作業系統 | Windows 10 / 11（64 位元） |
| 記憶體（RAM） | 8 GB 以上 |
| 磁碟空間 | 10 GB 以上可用空間 |
| 網路 | 首次安裝需要網路連線 |
| 帳號 | OpenAI 訂閱帳號（ChatGPT Plus 或 API 付費帳號） |

---

## 安裝步驟

### Step 1：安裝 Docker Desktop

1. 前往 [https://www.docker.com/products/docker-desktop/](https://www.docker.com/products/docker-desktop/)
2. 點選「Download for Windows」下載安裝程式
3. 執行下載的安裝程式，依照畫面指示完成安裝
4. 安裝完成後**重新開機**
5. 重新開機後，從桌面或開始選單開啟 **Docker Desktop**
6. 等待 Docker Desktop 完全啟動（底部狀態列顯示「Engine running」或鯨魚圖示停止動畫）

> 提示：Docker Desktop 首次啟動可能需要 2-5 分鐘，請耐心等待。

---

### Step 2：執行首次安裝（first-login.bat）

**僅需執行一次**

1. 確認 Docker Desktop 已完全啟動
2. 在 `docker` 資料夾中，**雙擊** `first-login.bat`
3. 如果出現「Windows 已保護您的電腦」提示，點選「**更多資訊**」→「**仍要執行**」
4. 等待程式自動下載必要檔案（視網路速度約需 5-15 分鐘）
5. 當程式提示「即將開啟 OpenAI 授權頁面」時，按任意鍵繼續
6. 在開啟的瀏覽器中，使用您的 **OpenAI 帳號登入並授權**
7. 登入成功後返回黑色視窗，等待提示「首次安裝設定完成」
8. 按任意鍵關閉視窗

---

### Step 3：日常啟動（start.bat）

**每次使用時執行**

1. 確認 Docker Desktop 已完全啟動
2. 在 `docker` 資料夾中，**雙擊** `start.bat`
3. 等待程式顯示「系統已就緒」
4. 瀏覽器會自動開啟 `http://localhost:3000`
5. 即可開始使用建安 AI 系統

---

## 注意事項

- **OpenAI 訂閱帳號**：使用 Codex AI 功能需要 ChatGPT Plus 訂閱或 OpenAI API 付費帳號，請事先確認帳號狀態
- **首次啟動需要網路**：`first-login.bat` 執行時會從網路下載必要的程式元件，請確保網路連線穩定
- **每次使用前開啟 Docker Desktop**：建安 AI 系統依賴 Docker 運行，請在執行 `start.bat` 之前先確認 Docker Desktop 已啟動
- **資料儲存位置**：您的資料會儲存在 `%USERPROFILE%\建安AI\data\` 資料夾（例如 `C:\Users\您的名稱\建安AI\data\`）

## 疑難排解

### 問題：執行腳本時顯示「Docker Desktop 尚未啟動」

**解決方法：**
1. 開啟 Docker Desktop 應用程式
2. 等待底部狀態列顯示「Engine running」
3. 再次執行 `start.bat`

---

### 問題：Port 3000 被占用（顯示「系統啟動超時」）

**解決方法：**

1. 按 `Windows + R`，輸入 `cmd`，按 Enter 開啟命令提示字元
2. 輸入以下指令，找出佔用 port 3000 的程式：
   ```
   netstat -ano | findstr :3000
   ```
3. 記下最右側的 PID 數字（例如 `12345`）
4. 輸入以下指令結束該程式（將 `12345` 換成您查到的 PID）：
   ```
   taskkill /PID 12345 /F
   ```
5. 重新執行 `start.bat`

---

### 問題：畫面出現亂碼（中文無法正常顯示）

**解決方法：**

在黑色視窗（命令提示字元）的標題列上按右鍵 → 「**內容**」→「**字型**」→ 選擇「**新細明體**」或「**微軟正黑體**」，按確定後重新執行腳本。

---

### 問題：docker compose pull 下載失敗

**解決方法：**
1. 確認網路連線正常，可嘗試開啟 [https://hub.docker.com](https://hub.docker.com) 確認是否可存取
2. 關閉 VPN 後重試
3. 重新執行 `first-login.bat`

---
