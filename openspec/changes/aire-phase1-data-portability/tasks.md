- [x] [P] 1.1 在 `src-tauri/src/crypto/master_password.rs` 實作 Argon2id 派生函式（Decision 1: 主密碼派生採 Argon2id（非 scrypt / PBKDF2））。行為：給定主密碼 + 16-byte salt 回傳 32-byte 派生 key；使用 OWASP 推薦參數 m_cost=19456, t_cost=2, p_cost=1。同時實作將 `argon2_params` 物件寫入 `keystore.json`（Argon2id parameters SHALL be persisted alongside salts）。對應 Requirement: System SHALL derive a key wrap with Argon2id OWASP parameters。驗證：`cargo test --package aire_core crypto::master_password::tests::derive_deterministic` 通過（同密碼+同 salt 兩次派生 byte-for-byte 相同）+ `derive_different_salts_yield_different_keys` 通過。
- [x] [P] 1.2 在 `src-tauri/src/crypto/recovery_code.rs` 實作 BIP39 救援碼生成、字串解析、wordlist 驗證（Decision 3: 救援碼採 BIP39 wordlist 12 個英文單字（非數字 / Base32））。行為：`generate()` 從 CSPRNG 取 128 bits 編出 12 個 BIP39 英文小寫單字；`parse(input: &str)` 做 lowercase / trim / split whitespace、回 `Result<[Word; 12], InvalidRecoveryCodeFormat>`，遇 token 數錯或 token 不在 wordlist 回錯誤並標出第一個無效 token。對應 Requirement: System SHALL generate 12 BIP39 English words on first-time master password setup + Recovery code input SHALL validate against BIP39 wordlist before deriving。驗證：`cargo test --package aire_core crypto::recovery_code::tests` 全綠（含「12 字輸出 / 全部在 wordlist / 11 字 reject / 13 字 reject / 非 BIP39 字 reject」5 個 case）。
- [x] 1.3 在 `src-tauri/src/crypto/vault.rs` 實作雙路 vault 設計（Decision 4: 救援碼派生第二把 key、雙路解鎖（非「救援碼解出主密碼」））。行為：`create_vaults(master_password, recovery_code) -> KeystoreFile` 生成 32-byte 隨機 sqlcipher_key、用 master 派生 key + AES-256-GCM 包成 vault_master、用 recovery 派生 key + AES-256-GCM 包成 vault_recovery；`unlock_with_master(password) -> Result<SqlcipherKey>` 與 `unlock_with_recovery(code) -> Result<SqlcipherKey>` 兩條路。對應 Requirement: SQLCipher key SHALL be a random 32-byte value wrapped twice + Recovery code SHALL derive a separate vault key with Argon2id。驗證：`cargo test crypto::vault::tests::both_paths_unlock_same_key` 通過（兩條路解出同一 32-byte key）+ `wrong_master_returns_aead_error` + `wrong_recovery_returns_aead_error` 通過。
- [x] 1.4 在 `src-tauri/src/crypto/master_password.rs` 加 `unlock(password)` 與 `set_first_time(password)` 包含最小長度與 zeroize：拒絕 < 8 code points 並回 `SetupError::PasswordTooShort`、派生完立即 `Zeroizing` 緩衝區。對應 Requirement: Unlock SHALL succeed with the correct master password + First-time master password SHALL enforce minimum length + System SHALL never persist the master password + System SHALL never persist the recovery code。驗證：`cargo test crypto::master_password::tests::short_password_rejected` 通過 + `cargo test crypto::master_password::tests::password_buffer_zeroized_after_derive` 用 `Zeroizing` 包裝後確認 dropped 前內容為 0 bytes。

## 2. `.aire` 檔格式 + 匯出 / 匯入後端

- [x] [P] 2.1 在 `src-tauri/src/data_portability/aire_format.rs` 實作 `.aire` ZIP 封包與解包（Decision 2: `.aire` 檔格式採 ZIP 容器（非自訂二進位格式））。行為：`pack(db_bytes, meta) -> Vec<u8>` 寫成 ZIP 含 `db.sqlite` + `meta.json` + `checksum.sha256`；`unpack(bytes) -> Result<UnpackedAire>` 反向解開；checksum 計算 = SHA-256(`db.sqlite` || `meta.json`)。對應 Requirement: System SHALL produce a `.aire` ZIP container on export。驗證：`cargo test data_portability::aire_format::tests::round_trip` 通過（pack 完 unpack 回得到 byte-for-byte 相同的 db_bytes 與 meta）+ `cargo test data_portability::aire_format::tests::container_has_three_entries` 確認 ZIP entries 為 `["db.sqlite","meta.json","checksum.sha256"]`。
- [x] [P] 2.2 在 `src-tauri/src/data_portability/export.rs` 實作 `export_aire(scope, output_path)` 含 SQLite IMMEDIATE 寫鎖與磁碟空間檢查。行為：(a) 開 `BEGIN IMMEDIATE` 取寫鎖，(b) byte-for-byte 複製 `db.sqlite` 到記憶體 buffer，(c) `ROLLBACK` 釋鎖，(d) pack 成 .aire 寫到 output_path；寫入前檢查 free space < db_bytes + 64 KiB 則回 `ExportError::InsufficientDiskSpace`，附 required vs available bytes 給 UI。對應 Requirement: Export SHALL acquire a write lock to prevent corruption + Export SHALL fail loudly when disk space is insufficient。驗證：`cargo test data_portability::export::tests::concurrent_write_blocked_during_export` 通過（測試在 export 進行中啟動的 INSERT 任務會被阻塞直到 ROLLBACK）+ `insufficient_disk_space_aborts` 通過（mock fs 回傳 free=10 bytes 時得到正確 error）。
- [x] 2.3 在 `src-tauri/src/data_portability/import.rs` 實作 `import_aire(file_path, master_password)` 入口含 checksum 與 schema 版本驗證。行為：unpack `.aire` → 重算 SHA-256 比對 `checksum.sha256` 不符回 `ImportError::CorruptedFile`；讀 `meta.json.schema_version` > 本機 `CURRENT_SCHEMA_VERSION` 回 `ImportError::IncompatibleSchema`；都通過後解 master_password 取得 sqlcipher_key 並把 db.sqlite 載入暫存 SQLite 連線供下一步衝突偵測使用。對應 Requirement: Import SHALL verify checksum and schema before unpacking。驗證：`cargo test data_portability::import::tests::corrupted_checksum_rejected` 通過（手動翻動 1 byte 觸發 CorruptedFile）+ `newer_schema_version_rejected` 通過。
- [x] 2.4 在 `src-tauri/src/data_portability/conflict.rs` 實作 case_id 衝突偵測與逐筆使用者選擇（Decision 5: 匯入衝突採「逐筆使用者選擇」（非全域策略））。行為：`detect_conflicts(local_db, imported_db) -> Vec<ConflictItem>` 比對 case_id 重疊；`resolve(item, action)` 套用 Overwrite / KeepNewer / Skip 三種行為（KeepNewer 比 `updated_at`）；session 層級 `apply_to_all(action)` 自動套用剩餘衝突。對應 Requirement: Import SHALL detect case_id conflicts and prompt the user per item。驗證：`cargo test data_portability::conflict::tests::three_conflicts_three_actions` 通過（3 衝突分別 Overwrite / KeepNewer / Skip 後的最終 row 內容符合預期）+ `apply_to_all_skip_propagates` 通過。
