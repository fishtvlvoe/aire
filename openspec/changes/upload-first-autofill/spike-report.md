## 假設 A：pdfjs-dist 能從真實謄本抽出中文文字層

**狀態**：✅ PASS
**方法**：`pdfjs.getDocument({ data, useSystemFonts: true }).getTextContent()`

### 結果

- 頁數：2（page 1 = 土地謄本 / page 2 = 建物謄本，單一 PDF 合併兩份）
- 有無文字層：**有**（非掃描影像）
- 總字數：1975（page 1: 982 / page 2: 993）
- 中文可讀性：**100% 可讀，無亂碼**
- 原始輸出節錄（page 1 前 80 字）：
  > 土地登記第一類謄本（所有權個人全部）永康區大灣段 6338-0000地號列印時間：民國115年04月02日17時13分

### 結論

Layer 1（pdfjs 文字層）對電子謄本完全可行。Phase 1 不需改走 OCR。

**額外發現**：
- 謄本大量使用 `*` 填充字元對齊（例：`權利範圍：*****10000分之91********`），parser 必須先 strip `*`
- 一份 PDF 可能同時包含土地 + 建物謄本兩頁 → **pipeline 需要「頁面分類」步驟**，不能假設一個檔案一種類型
- 金額/面積以全形半形混用（例：`35,000.0元／平方公尺`，全形斜線），regex 要注意

## 假設 B：5 行 regex 抽 4 核心欄位，命中 ≥ 3/4

**狀態**：✅ PASS（8 個欄位命中 7 個，87.5%）
**方法**：對 `getTextContent()` 輸出直接跑 regex，無預處理。

### Regex 版本 v1
```js
const regex = {
  section: /([\u4e00-\u9fa5]+段)\s*(\d{4,5}-\d{3,4})(?:地號|建號)/,
  area:    /(?:面\s*積|總面積)：\*+([\d,]+\.\d+)\s*平方公尺/,
  rights:  /權利範圍：\**([^\n*]+?)\*+/,
};
```

### 命中結果

| 頁 | 欄位 | 抽出值 | 正確值（人工） | 命中 |
|----|------|--------|--------------|------|
| 1（土地） | 地段 | 永康區大灣段 | 永康區大灣段 | ✅ |
| 1（土地） | 地號 | 6338-0000 | 6338-0000 | ✅ |
| 1（土地） | 面積 | 1,223.00 平方公尺 | 1,223.00 平方公尺 | ✅ |
| 1（土地） | 權利範圍 | 10000分之91 | 10000分之91 | ✅ |
| 2（建物） | 地段 | 永康區大灣段 | 永康區大灣段 | ✅ |
| 2（建物） | 建號 | 07615-000 | 07615-000 | ✅ |
| 2（建物） | 面積 | 84.13 平方公尺 | 84.13 平方公尺 | ✅ |
| 2（建物） | 權利範圍 | （空） | 全部（1分之1） | ❌ |

**命中率：7/8 = 87.5%**

### 結論

Regex 路線可行。未命中的 1 個是格式變異：
- Pattern A：`權利範圍：*****10000分之91********`（純分數）→ v1 regex 能抓
- Pattern B：`權利範圍：全部 *********1分之1*********`（文字 + 分數混合）→ v1 regex miss

Phase 1.4/1.5 parser 需擴充處理 Pattern B，並把「全部」「1分之1」分別 normalize 成 `rights.text = "全部"` + `rights.fraction = "1/1"`。

### 微幅補強（寫入 tasks.md Phase 1）

1. **新增前置步驟**：Task 1.3 抽文字後先做 `stripFillerChars(text)` — 去除 `*` 填充與全形空白。
2. **新增頁面分類**：Task 1.3 與 1.4/1.5 之間加「headerDetect」— 依「土地登記第一類謄本」/「建物登記第一類謄本」分類，再送對應 parser。
3. **Task 1.4 權利範圍** regex 必須同時處理兩種 pattern（純分數 vs 文字+分數）。

這 3 點不改 Phase 結構、不延長時程，只是 implementation 細節補強。Phase 2-6 完全不動。
