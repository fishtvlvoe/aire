## 5. Parser 設計建議

### Parser 數量
建議 **3 種**：LandParser（土地標示+所有權+他項）、BuildingParser（建物標示+所有權+他項）、MixedParser（兩者組合）。

### Section 根型別
| Section | 根型別 |
|---------|--------|
| 土地他項權利部 | list |
| 土地所有權部 | list |
| 土地標示部 | dict |
| 建物他項權利部 | list |
| 建物所有權部 | list |
| 建物標示部 | dict |

### Normalize 建議
| 欄位 | 方式 | 輸出型別 |
|------|------|---------|
| 面積（土地/建物） | 移除 `,`+`平方公尺` → float | float |
| 公告土地現值 / 公告地價 / 當期申報地價 | 移除 `,`+`元/平方公尺` → int | int |
| 登記日期 / 建築完成日期 / 原因發生日期 | `民國YYY年MM月DD日` → `YYYY-MM-DD` | date |
| 公告地價年月 / 公告現值年月 | `民國YYY年MM月` → `YYYY-MM` | year-month |
| 地號 / 建號 | 拆為 `{地段, 地號/建號}` + 保留原始字串 | object |
| 建物門牌 | 視為坐落地址，保留原字串 | string |
| 權利範圍 | `全部1分之1`→`1/1`；`X分之Y`→`Y/X` | string |
| 使用分區 / 使用地類別 | `(空白)` → null | null |
| 其他登記事項 / 地價備註事項 | `(空白)` → `[]` | list[string] |
| 層數 | `010層` → `10`（去前導零） | int |
| 層次 | list[{層次, 層次面積}]，面積同上面積 normalize | list[object] |
| 附屬建物 | list[{用途, 面積}] | list[object] |
| 共有部分 | list[{建號, 權利範圍}] | list[object] |
| 建物座落地號 | list[string] | list[string] |
| 地上建物建號 | list[string] | list[string] |
