## 1. 定期備份資料

定期備份可確保電腦故障或系統崩潰時資料不遺失。

**操作步驟：**

1. 開啟 AIRE → 點選右上角齒輪圖示 → 進入「設定」
2. 選擇「資料管理」→「匯出備份」
3. 輸入主密碼確認身份
4. 選擇儲存位置（建議選擇外接硬碟或雲端同步資料夾）
5. 確認產生 `.aire` 備份檔（檔名格式：`aire-backup-YYYYMMDD.aire`）

**建議頻率：** 每週至少一次，或每完成重要案件後立即備份。

> **注意：** `.aire` 備份檔已加密，遺失主密碼或救援碼將無法開啟。

## 2. 換電腦匯入資料

從舊電腦遷移至新電腦時，請依下列步驟操作。

**前置準備：**
- 確認已有最新的 `.aire` 備份檔
- 確認記得主密碼或持有救援碼

**操作步驟：**

1. 在新電腦安裝 AIRE（從官方網站下載最新版本）
2. 啟動 AIRE → 選擇「從備份還原」
3. 選取 `.aire` 備份檔案
4. 輸入主密碼（或使用救援碼）解鎖
5. 等待匯入完成 → 確認案件清單與舊電腦一致

**匯入完成後請確認：**
- [ ] 案件總數與備份前相符
- [ ] 每筆案件資料（屋主、地址、揭露項目）完整
- [ ] 授權資訊已正確載入

## 3. 忘記主密碼

若無法記起主密碼，可透過救援碼重設。

> **⚠️ 警告：沒有救援碼則無法重設主密碼，請先確認手邊有救援碼再進行操作。**

**操作步驟：**

1. 開啟 AIRE → 在解鎖畫面點選「忘記密碼？」
2. 選擇「使用救援碼重設」
3. 輸入完整的 24 字元救援碼（格式：`XXXX-XXXX-XXXX-XXXX-XXXX-XXXX`）
4. 系統驗證後，輸入新主密碼（至少 8 字元，建議包含大小寫與數字）
5. 再次輸入新主密碼確認
6. 點選「確認重設」→ 使用新密碼登入

**重設後建議：**
- 重新備份一份新的 `.aire` 檔
- 更新密碼管理器中儲存的主密碼記錄

## 4. 救援碼也遺失

> **⚠️ 嚴重警告：主密碼與救援碼同時遺失，所有資料將永久無法存取。**
> **AIRE 採用端對端加密設計，伺服器端不儲存任何解密金鑰，技術支援團隊無法協助還原。**

**此情況下唯一可行的處置方式：**

1. 卸載並重新安裝 AIRE（將清除所有本機加密資料）
2. 重新啟動全新帳號與授權
3. 所有案件資料需手動重新建立

**強烈建議立即執行預防措施（見下方保管建議），避免再次發生。**

## 救援碼保管建議

> **救援碼只會在設定主密碼時顯示一次，請立即以下列方式保存：**

### 建議一：密碼管理器

使用 1Password、Bitwarden 或 Apple 鑰匙圈儲存救援碼。
- 優點：跨裝置同步、不怕遺失紙本
- 操作：建立新項目 → 類別選「安全備忘」→ 貼上救援碼

### 建議二：實體紙本存放保險箱

將救援碼列印或手寫後，存放於保險箱或上鎖的抽屜。
- 優點：不受網路或帳號狀態影響
- 注意：避免放在容易被他人看見的地方

### 建議三：加密雲端文件

使用加密保護的雲端文件（如 Notion 加密頁面、加密 PDF）儲存。
- 優點：易於存取，適合行動工作者
- 注意：確保雲端帳號本身有 2FA 保護

## 資料衝突處理

當同一份備份在兩台電腦各自修改後，重新合併可能產生衝突。

**三種衝突情境與對應動作：**

### 情境一：同一案件在兩台電腦分別編輯

**範例：** 「台北信義路物件」在 A 電腦更新了屋況說明，在 B 電腦更新了屋主聯絡資料。

**對應動作：**
1. 系統會標示「衝突版本」並顯示差異比對畫面
2. 選擇「保留 A 電腦版本」、「保留 B 電腦版本」或「手動合併」
3. 確認後系統產生合併後的統一版本

### 情境二：其中一台電腦已刪除案件，另一台仍保有

**範例：** A 電腦刪除「新北板橋物件」，B 電腦仍有該案件資料。

**對應動作：**
1. 系統詢問「此案件在另一裝置已刪除，是否還原？」
2. 選擇「還原案件」（保留 B 的版本）或「確認刪除」（以 A 的動作為準）
3. 選擇後系統同步兩端狀態

### 情境三：備份檔版本比目前資料庫舊

**範例：** 使用三個月前的備份檔進行還原，但目前資料庫已有更新的案件。

**對應動作：**
1. 系統警示「備份檔較舊，部分資料將遺失」並列出差異清單
2. 選擇「僅還原備份中缺少的案件」（合併模式）或「完整覆蓋目前資料」（還原模式）
3. 強烈建議選擇「合併模式」以保留最新資料
