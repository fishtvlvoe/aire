# 需求摘要（來源：我想要的架構.docx）

## 系統核心流程

```
房屋資料輸入 → 區域資料分析 → 市場行情分析 → AI 策略生成 → 多種內容輸出
```

## 輸入欄位

| 欄位 | 說明 |
|------|------|
| 地址 | 房屋所在地址 |
| 地段地號 | 土地登記資訊 |
| 開價 | 屋主期望售價 |
| 坪數 | 室內面積 |
| 屋齡 | 房屋年齡 |
| 格局 | 幾房幾廳幾衛 |
| 樓層 | 所在樓層 |
| 車位 | 有無車位 |
| 周邊設施 | 生活機能描述 |

## 9 大輸出項目

| # | 輸出項目 | 類型 | 用途 |
|---|----------|------|------|
| 1 | 不動產行銷優劣分析（SWOT） | 分析報告 | 內部策略參考 |
| 2 | 行銷文案（3 種風格） | 文字內容 | 社群/廣告投放 |
| 3 | 經營屋主指南 | 策略文件 | 業務員與屋主溝通 |
| 4 | 市場預期價格 | 數據分析 | 定價/議價依據 |
| 5 | 議價策略 | 策略文件 | 買方談判使用 |
| 6 | 行銷影片劇本 | 創意內容 | 影片製作 |
| 7 | 分鏡腳本 | 創意內容 | 影片製作 |
| 8 | AI 影片生成提示詞 | AI prompt | 餵給 AI 影片工具 |
| 9 | 行銷歌曲 + MV 腳本 | 創意內容 | 品牌行銷 |

## 原始文件建議的技術架構

- 前端：Web 表單
- 後端：AI API
- 外部資料：Google Maps API + 房價資料
- AI 引擎：GPT 分析
- 輸出：結構化報告

## 範例場景

文件以「台中七期 / 3280 萬 / 45 坪 / 5 年 / 3 房 2 廳」為例，展示了完整的 9 項輸出範例。
