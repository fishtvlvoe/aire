## 能用的管道（詳細）

### 1. Google Maps API — 查周邊設施

可查：學校、醫院、公園、車站、餐廳、銀行、警局等

| API | 免費額度 | 超過後費用 |
|-----|---------|-----------|
| Geocoding（地址→座標） | 10,000 次/月 | $5/千次 |
| Nearby Search（附近設施） | 10,000 次/月 | $32/千次 |
| Place Details（設施詳情） | 10,000 次/月 | $5/千次 |

每月免費額度可查約 2,000 個地址的附近設施。客戶一個月處理 10-20 個物件，**完全在免費額度內**。

### 2. 實價登錄 Open Data — 查歷史成交

已驗證可行：
```
curl https://plvr.land.moi.gov.tw/DownloadSeason?season=113S3&type=a&fileName=lvr_landcsv.zip
→ 成功下載 20MB ZIP
```

- 每月 1/11/21 日更新
- CSV 含 33 個欄位（地址、總價、單價、坪數、屋齡、格局等）
- 按縣市分檔（a=臺北、b=新北...）
- **策略：排程下載 → 匯入資料庫 → 用地址模糊比對查詢**

### 3. 國土測繪中心 API — 查行政區/地段

免費 REST API，回傳 XML：
- `GET api.nlsc.gov.tw/other/ListTown/D` → 臺南市所有行政區
- `GET api.nlsc.gov.tw/other/ListLandSection/D/D01` → 東區所有地段
- `GET api.nlsc.gov.tw/other/TownVillagePointQuery/{lng}/{lat}` → 座標→行政區

**限制：門牌轉地號的 API 限政府機關，民間無法申請**

## 不能用的管道（和替代方案）

### 電子謄本 — 法律明確禁止自動化

> 使用條款第七條：禁止「透過自動化程式進行資料自動擷取」

替代方案：
- 免費的「他項權利人查詢」按鈕 → 只能手動操作
- 客戶自己查完拍照/截圖丟進系統

### 門牌轉地號 — 沒有免費程式化管道

| 方式 | 費用 | 可行性 |
|------|------|--------|
| e 點通 | $10/筆 | 需憑證 |
| twland.ronny.tw | 免費 | 資料停在 2015 年，過時 |
| 國土測繪中心地籍 API | 免費 | 限政府機關 |

替代方案：**用實價登錄 CSV 反查**。CSV 裡有門牌地址，可以用地址模糊比對找到對應的地段地號。

### 凶宅 — 做不到

Cloudflare 防護 + 非結構化論壇資料。
替代方案：系統不查凶宅，明確列在「未檢查項目」。

### 建物使用執照 — 臺南市系統連不上

需要打電話確認：06-2991111（臺南市府）

## 對產品架構的影響

### 「輸入地址自動查」能做到什麼程度？

```
客戶輸入地址
  ↓
1. Google Maps Geocoding → 取得經緯度座標        ✅ 自動
2. 國土測繪中心 API → 確認行政區/地段             ✅ 自動
3. 實價登錄資料庫 → 查同區域歷史成交              ✅ 自動
4. Google Maps Nearby Search → 查周邊設施         ✅ 自動
5. 公告現值 → 查該地段公告地價                    ⚠️ 可能自動（不穩定）
6. 都市計畫分區 → 查使用分區                      ⚠️ 需實測
7. 他項權利（抵押/假扣押） → 需客戶手動查          ❌ 不能自動
8. 凶宅/違建/海砂屋 → 系統不查                    ❌ 不能自動
```

### 結論：6 成自動 + 4 成手動/不查

**自動的部分已經夠用。** 核心需求（實價登錄 + 周邊設施 + 行政區定位）都能自動化。

他項權利和凶宅必須手動，但這本來就是客戶自己要判斷的。
