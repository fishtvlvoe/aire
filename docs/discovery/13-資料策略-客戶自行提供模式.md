# 資料策略：「客戶自行提供 + 系統分析」模式

> 日期：2026-03-19
> 狀態：討論中

## 核心概念

```
❌ 不做：系統自動爬蟲（違法風險）
✅ 要做：客戶自己查到資料 → 丟進系統 → AI 分析
```

系統是「資料分析引擎」，不是「資料爬蟲」。

## 資料流設計

| 資料來源 | 誰取得 | 怎麼進系統 | 系統角色 |
|----------|--------|-----------|---------|
| 實價登錄歷史成交 | 系統自動 | 排程下載 CSV（每 10 天） | 主動抓取（合法免費） |
| 591 同區開價 | 客戶自己查 | 截圖或貼文字 | 被動接收 → AI 辨識分析 |
| 永慶/信義/其他平台 | 客戶自己查 | 截圖或貼文字 | 被動接收 → AI 辨識分析 |
| 謄本 | 客戶自己取得 | 拍照上傳 | AI OCR 辨識 |
| 他項權利（抵押/假扣押） | 系統查（免費）或客戶拍謄本 | 自動查詢或照片 | 主動查詢 + AI 辨識 |
| 客戶自己的經驗判斷 | 客戶自己 | 文字輸入 | 納入 AI 分析 |

## 這樣做的好處

1. **完全合法** — 資料是客戶自己查的，系統只是幫他分析
2. **不限平台** — 591、永慶、信義、好房網...任何網站的資料都可以丟進來
3. **資料更即時** — 客戶現場查的就是最新資料，不用等 API 更新
4. **降低開發成本** — 不用維護爬蟲、不用處理反爬機制

## 使用場景

```
客戶遇到新物件
  → 拍謄本上傳 → AI 辨識基本資料 + 他項權利
  → 系統自動查實價登錄同區域歷史成交
  → 客戶打開 591 查同區目前開價 → 截圖丟進系統
  → AI 綜合分析：
     「實價登錄近 3 個月成交均價 850 萬
      你提供的 591 同區開價中位數 920 萬
      此物件開價 1000 萬，高於行情約 8-17%
      建議出價區間：830-900 萬」
```

## 要解決的 UX 問題

客戶要自己去查 591 再貼進來，這會不會太麻煩？

| 做法 | 操作步驟 | 便利度 |
|------|---------|--------|
| 截圖上傳 | 開 591 → 截圖 → 回到系統貼上 | ⭐⭐ 尚可 |
| 複製網址 | 開 591 → 複製物件網址 → 貼進系統 | ⭐⭐⭐ 方便（但系統要能讀取網頁） |
| 手動輸入 | 看 591 → 在系統裡輸入「同區 3 間，開價分別是...」 | ⭐ 麻煩 |
| 不提供 591 資料 | 純用實價登錄分析 | ⭐⭐⭐⭐ 最省事（但少了即時行情） |

> 待問客戶：「你願意多花 1 分鐘去查 591 再截圖丟進來嗎？還是只用實價登錄的資料就夠了？」
