# 產品規劃 — 兩條產品線與開發排程

> 日期：2026-03-19
> 狀態：已確認

## 產品架構

同一個 Web App，功能分階段上線：

```
同一個網站
├── Phase 1：物件分析助手（產品線 A）→ 2 個月 → 10 萬
└── Phase 2：行銷自動化（產品線 B）→ 另議
```

## 產品線 A：物件分析助手（MVP）

### 功能清單

| # | 功能 | 說明 |
|---|------|------|
| A1 | 地址自動查詢 | 輸入地址 → 系統自動從線上查詢所有可取得的資料 |
| A2 | 拍照辨識（備用） | 線上查不到時，拍謄本讓 AI 辨識 |
| A3 | 手動輸入（最後手段） | 以上都不行時手動填寫 |
| A4 | 實價登錄分析 | 自動查詢同區域歷史成交，計算均價、中位數、單價 |
| A5 | 市場價格預估 | 根據實價登錄 + 物件條件，給出建議售價區間 |
| A6 | 議價策略 | 根據開價 vs 行情落差，產出議價建議和話術 |
| A7 | 不動產優劣分析 | 物件 SWOT：賣點、缺點、機會、威脅 |
| A8 | 屋主經營指南 | 簽委託後跟屋主的溝通策略 |
| A9 | 物件風險檢查 | 查詢他項權利（抵押/假扣押） |
| A10 | 客戶提供資料分析 | 客戶貼入 591 截圖或其他平台資料，AI 綜合分析 |
| A11 | 物件管理列表 | 所有分析過的物件一覽表 |

### 輸入方式優先級（修正版）

| 優先級 | 方式 | 場景 |
|--------|------|------|
| ⭐ 主要 | 輸入地址 → 系統自動查 | 大多數情況 |
| 備用 | 拍照辨識謄本 | 查不到或要看他項權利細節 |
| 最後手段 | 手動輸入 | 以上都不行時 |

### 明確排除（不含在產品線 A）

- ❌ 行銷文案生成 → 產品線 B
- ❌ 自動上架 591 / 房屋網 → 產品線 B
- ❌ 影片劇本 / 分鏡腳本 → 產品線 B
- ❌ AI 影片生成提示詞 → 產品線 B
- ❌ 歌曲 / MV 腳本 → 產品線 B
- ❌ 即時對話錄音分析 → 未來規劃
- ❌ 多使用者帳號管理 → SaaS 化時再做
- ❌ 手機原生 APP → RWD 網頁足夠
- ❌ 自動爬蟲 591 / 永慶 → 法律風險
- ❌ 正式不動產估價報告 → 需估價師資格

## 產品線 B：行銷自動化（Phase 2）

| # | 功能 | 說明 |
|---|------|------|
| B1 | 行銷文案生成 | 3 種風格一鍵產出 |
| B2 | 一鍵上架準備 | 系統整理好資料，客戶複製到 591 |
| B3 | 影片劇本 | 60 秒行銷影片劇本和分鏡 |
| B4 | AI 影片提示詞 | 每 15 秒一段的影片生成 prompt |
| B5 | 行銷歌曲 + MV | 歌詞、旋律描述、MV 分鏡 |

> 產品線 B 在 A 完成後另外報價，預估 6-10 萬

## 線上資料查詢管道（待技術驗證）

| 資料 | 查詢管道 | 只需要 |
|------|---------|--------|
| 建物/土地基本資訊 | easymap.land.moi.gov.tw | 地址 |
| 他項權利 | 全國地政電子謄本系統 | 地號 |
| 實價登錄 | lvr.land.moi.gov.tw + CSV 批次下載 | 地址/區域 |

| 都市計畫分區 | 各縣市都發局 | 地址/地號 |
| 建物使用執照 | 各縣市建管處 | 地址 |

> 以上管道需要實際測試哪些能用程式自動查詢
