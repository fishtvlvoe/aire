## 技術決策

| 項目 | 決定 | 理由 |
|------|------|------|
| 產品入口 | Web App（手機瀏覽器開） | LINE 限制太多、不做原生 App |
| 報告格式 | 類 ChatGPT 串流回覆 + 卡片式 | 即時更新、不需要正式 PDF |
| 照片辨識失敗 | 直接請客戶重拍 | 不做複雜的模糊修正 |
| 開發模式 | Claude 寫程式碼，你當架構師 | 跟 BuyGo/LineHub 同樣模式 |
| 技術方案 | 不用 n8n | 不想維護又一個複雜系統 |

## 報告呈現方式

像 Claude 的對話介面：
- 資料輸入後，系統像 AI 一樣一段一段產出分析
- 可以即時更新、追問
- 卡片式區塊呈現（風險、價格、策略各一塊）
- 不需要匯出 PDF
