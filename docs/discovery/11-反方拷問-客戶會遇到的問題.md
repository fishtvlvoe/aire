# 反方拷問 — 客戶會遇到的六大問題

> 日期：2026-03-19
> 角色：反方顧問（唱反調）

## 問題 1：AI 估的價格，跟現實可能差很多

- 實價登錄資料延遲 1-3 個月
- 客戶說房價在跌 → 3 個月前的成交價已經過時偏高
- AI 用過時資料估出「合理價」會偏高
- **最壞情況：客戶拿偏高數據跟屋主談，屋主以為值這個價，簽了高於行情的委託，賣不掉**

> 待問客戶：「你能接受系統有 5-10% 誤差嗎？還是期待跟你自己估的一樣準？」

## 問題 2：系統說「沒有風險」不代表真的沒風險

系統查得到的：✅ 抵押權、假扣押

系統查不到的：
- ❌ 凶宅（無政府 API）
- ❌ 違建（要比對使用執照 + 現場）
- ❌ 海砂屋、輻射屋（政府列管名冊無 API）
- ❌ 漏水、結構問題（要現場看）
- ❌ 嫌惡設施（地圖可查但不一定準）

**最壞情況：客戶看到「無異常」就簽委託，結果是凶宅**

> 待問客戶：「你現在判斷物件有沒有問題，除了看謄本還會做什麼？」

## 問題 3：分析變快了，但時間瓶頸沒解決

```
現在：20 物件 → 分析 5 個 → 簽 3 個 → 成交 1 個
用系統：20 物件 → 全分析 → 簽 10 個 → 還是成交 1-2 個
```

簽更多委託 ≠ 賣更多房子。帶看、談判、成交的時間沒變少。
8 個沒成交的屋主會覺得「你不積極」。

> 待問客戶：「你現在簽的委託成交率多少？簽更多但成交率不變，扛得住嗎？」

## 問題 4：屋主不看報告，報告的專業感有用嗎？

- 屋主看不到報告
- 客戶口頭說「根據系統分析...」，屋主會信嗎？
- 在房仲業，信任靠關係，不是靠 AI 報告

> 待問客戶：「你跟屋主談降價時，屋主比較信數據還是信你的經驗？」

## 問題 5：SaaS 賣給別人 = 賣掉他的競爭優勢

- 同區域的競爭對手也用同一套系統 → 他的系統優勢歸零

> 待問客戶：「如果你的同區域競爭對手也用了這套系統，你覺得有影響嗎？」

## 問題 6：地政 API 費用是未知數

- 查一次謄本的他項權利可能要 10-20 元
- 每月 100 次 = 1000-2000 元
- 會吃掉維運預算

> 待查：地政 API 實際收費（調研中）

## 各環節問題嚴重度

| 環節 | 嚴重度 | 說明 |
|------|--------|------|
| 實價登錄 | 🟡 中 | 可用但延遲 1-3 個月 |
| 地政 API | 🟡 中 | 可用但費用未知 |
| 591 資料 | 🔴 高 | 不能爬蟲，「目前開價」做不了 |
| AI 估價精度 | 🟡 中 | 台灣資料量有限 |
| 拍照辨識 | 🟢 低 | 可行 |
| 法律免責 | 🟡 中 | 必須加免責聲明 |
| 凶宅/違建 | 🔴 高 | 系統做不到 |

## 待問客戶完整清單（累計 22 題）

### A. 物件與資料（第二輪）
1. 拿到物件時手上通常有什麼文件？
2. 謄本是紙本還是電子檔？
3. 一天處理幾個新物件？
4. 操作區域主要在哪？

### B. 工作流程（第二輪）
5. 從接物件到簽委託的標準流程？每步花多久？
6. 判斷開價合不合理靠經驗還是查資料？查什麼？
7. 「黑箱測試」多常做？效果怎麼判斷？
8. 底下幾個員工？未來也要用嗎？

### C. 系統使用（第二輪）
9. 手機 iPhone 還是 Android？
10. 報告怎麼看？手機看？PDF？
11. 現在有在用什麼 AI 工具嗎？
12. 每次操作願意花幾分鐘？

### D. 拍照辨識（第四輪追加）
13. 謄本拍照是每頁都拍？還是只拍特定部分？
14. 需要系統辨識他項權利部嗎？還是自己看？

### E. 反方問題（第五輪追加）
15. 你自己估價的準確率多少？跟成交價差多遠？
16. 有沒有簽爛單賠錢的經驗？什麼情況？
17. 員工知道你要做這個系統嗎？會怎麼想？
18. 跟屋主談判時最常遇到的困難是什麼？
19. 能接受系統有 5-10% 誤差嗎？
20. 現在判斷物件有沒有問題，除了謄本還做什麼？
21. 簽的委託成交率大概多少？
22. 同區域對手也用這套系統，你覺得有影響嗎？
