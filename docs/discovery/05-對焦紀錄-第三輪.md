# 對焦紀錄 — 第三輪

> 日期：2026-03-19
> 狀態：已確認

## 關鍵決策確認

### 產品定位：先私人 → 未來 SaaS

- 客戶同意未來可以賣給其他人
- 現階段先做給客戶用（單租戶）
- 架構上要預留擴展空間，但不需要現在就做多租戶

### 物件風險檢查：要做，且要告知是什麼風險

- 不是只說「有風險」就好，要說明具體是什麼問題
- 例如：「此物件有第二順位抵押權，設定金額 XXX 萬」

### 即時對話分析：暫列為未來功能

- 是顧問的想法，不是客戶的硬需求
- 不放入 MVP，但納入產品路線圖

### 開發模式

- 不找工程師
- 顧問（你）= 架構師 + 產品經理
- Claude = AI 工程師
- 客戶 = 需求來源 + 測試者

### 協作流程

```
客戶口述需求
  → 你翻譯成 AI 能理解的邏輯架構
  → Claude 產出計畫 + 開發
  → 有問題 → 你帶電腦去找客戶，當面問
  → 客戶直接講，內容貼給 Claude
  → Claude 產出 PRD
  → 往下開發
```

### 競品現況

- 同業有用 AI 工具（剪輯、短影片等）
- 但**沒有人做「議價分析 + 物件風險評估」這類決策輔助工具**
- 我們要做的是：了解屋主的底牌，輔助房仲判斷

## 待問客戶的問題（更新版）

> 合併第二輪 + 第三輪，整理成一份完整清單

### A. 物件與資料

| # | 問題 | 目的 |
|---|------|------|
| 1 | 你拿到物件時，手上通常有什麼文件？（謄本、權狀、名片、DM、截圖？） | 決定 AI 辨識格式 |
