> 日期：2026-03-19
> 狀態：已確認

## 客戶畫像

| 項目 | 內容 |
|------|------|
| 身份 | 房仲公司老闆，兼業務員 |
| 痛點 | 手頭物件多，沒時間逐一分析，無法判斷屋主開價是否合理 |
| 現況 | 手動處理 + 一位助理，但時間不夠 |
| 使用場景 | **簽委託前**，用系統產出的數據說服屋主調整開價 |
| 操作方式 | 手機端 + Web 端，SaaS 服務 |

## 核心使用情境

```
拍照輸入物件資料
    → AI 辨識並對應區域
    → 自動調閱實價登錄 + 同區域公開開價
    → 產出分析報告
    → 房仲拿著報告跟屋主談：
      「開 1000 萬可能要賣一年，
       出在 850-950 萬成交機率高很多」
    → 屋主同意 → 簽委託
```

## 四大核心輸出（優先級已確認）

| 優先級 | 輸出 | 為什麼重要 |
|--------|------|-----------|
| P0 | 議價策略 | **最重要** — 直接影響能不能簽到委託 |
| P0 | 市場預期價格 | 議價的數據基礎，要有根據才能說服 |
| P1 | 不動產行銷優劣分析 | 讓屋主知道自己物件的定位 |
| P1 | 屋主經營指南 | 簽委託後的溝通策略 |
| P2 | 文案/影片/歌曲（5 項） | 行銷加值，非核心痛點 |
