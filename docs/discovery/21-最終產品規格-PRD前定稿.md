# 最終產品規格 — PRD 前定稿

> 日期：2026-03-19
> 狀態：已確認，準備進入 PRD 撰寫

## 產品定位

**不動產物件分析助手** — 減少房仲調研工作量的 AI 工具。
不是 AI 賣房系統，不取代房仲判斷，是放大鏡讓房仲看得更清楚。

## 使用者

- 房仲公司老闆（兼業務員）
- 單一使用者（MVP 不做多帳號）
- 手機 + 電腦操作（RWD Web App）

## 核心流程

```
輸入地址（一個欄位）
  ↓ 全自動（3 分鐘內）
  ├── 座標定位 + 行政區確認
  ├── 實價登錄分析（3 公里內，近 1 年）
  ├── 周邊設施分析（3 公里內）
  ├── 市場價格預估 + 議價策略
  ├── SWOT 分析 + 屋主經營指南
  └── 自動生成 5 個平台的搜尋連結
  ↓
  初版報告產出
  ↓ 客戶選擇性補充
  ├── 點連結查 591 等平台 → 截圖傳回 → AI 更新分析
  ├── 輸入他項權利查詢結果 → 風險標示
  └── 輸入自己的經驗判斷 → AI 納入考量
  ↓
  最終報告
```

## 功能清單（產品線 A：MVP）

### 自動產出（客戶只打一個地址）

| # | 功能 | 內容 |
|---|------|------|
| A1 | 物件基本資訊卡 | 座標、行政區、里、地段 |
| A2 | 實價登錄分析 | 3km 內近 1 年成交筆數、均價、中位數、每坪單價、趨勢、原始紀錄、信心度 |
| A3 | 周邊設施分析 | 最近學校/醫院/公園/車站/超市/警局 + 距離 + 生活機能評分 |
| A4 | 市場價格預估 | 建議售價區間、開價 vs 行情落差%、預估銷售時間 |
| A5 | 議價策略 | 開價合理性、建議議價空間、3 種話術、買方心理、堅持開價的風險 |
| A6 | SWOT 分析 | 優勢/劣勢/機會/威脅 + 整體評估 |
| A7 | 屋主經營指南 | 溝通策略、回報頻率、降價時機、避免事項 |
| A8 | 平台搜尋連結 | 自動生成 591/好房網/永慶/信義/Google Maps 的搜尋連結（3km 範圍） |

### 客戶手動補充後更新

| # | 功能 | 內容 |
|---|------|------|
| A9 | 591/平台資料分析 | 客戶截圖或輸入同區開價 → AI 比較分析 |
| A10 | 物件風險檢查 | 客戶輸入抵押/假扣押結果 → 風險等級標示 |
| A11 | 經驗值分析 | 客戶輸入自己的判斷 → AI 納入考量 |

### 管理功能

| # | 功能 | 內容 |
|---|------|------|
| A12 | 物件管理列表 | 所有物件一覽、狀態標記、歷史報告、搜尋篩選 |
| A13 | 手動輸入 | 自動查詢失敗時的備用輸入方式 |
| A14 | 拍照辨識 | 拍謄本讓 AI 辨識（備用方案） |

### 系統內建

| # | 功能 | 內容 |
|---|------|------|
| S1 | 實價登錄自動更新 | 每月 1/11/21 日排程下載 CSV |
| S2 | 數字來源驗證 | 每個數字可點擊展開來源紀錄 |
| S3 | 信心度分級 | ≥20 筆高信心 / 10-19 有限 / 5-9 警告 / <5 不給估價 |
| S4 | 免責聲明 | 「市場參考分析，非正式估價報告」 |
| S5 | 資料時間標示 | 每份報告標示資料更新日期 |

## 技術架構

```
前端：Next.js（RWD，類 ChatGPT 介面）
後端：Next.js API Routes 或 Python FastAPI
資料庫：PostgreSQL（Supabase）
AI：Claude API（分析引擎）
外部 API：
  ├── Google Geocoding（地址→座標）免費 10,000 次/月
  ├── Google Nearby Search（周邊設施）免費 10,000 次/月
  └── 國土測繪中心 API（行政區/地段）免費
資料源：
  └── 實價登錄 Open Data CSV（排程下載，免費）
部署：Vercel（serverless，幾乎免維護）
```
