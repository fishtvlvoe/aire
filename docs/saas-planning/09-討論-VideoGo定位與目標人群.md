# 討論：VideoGo 定位與目標人群

## 第 1 輪

### 用戶提出

VideoGo 的核心價值是幫用戶省去摸索時間。目前 AI 影像平台大多是英文介面，VideoGo 可以引導用戶一步步產出內容（提示詞 → 分鏡 → 生成）。

類比：VideoGo 就像 IDE，底層 API（Kling/fal.ai）是編譯器，我們提供引導、範本、專案管理。

### 分析：影片生成 IDE 定位

| 層次 | 原始工具（fal.ai / Kling） | VideoGo |
|------|---------------------------|---------|
| 介面 | 英文、通用型 | 中文、針對影片內容創作 |
| 流程 | 用戶自己摸索 prompt | 引導式：角色包 → 分鏡 → 生成 |
| 資產管理 | 沒有 | 角色包、定妝照、Element Binding |
| 學習曲線 | 高（要懂 prompt engineering） | 低（系統幫你組 prompt） |

### 策略共識

| 我們做 | 我們不做 |
|--------|---------|
| 引導式工作流（prompt → 分鏡 → 生成） | 模型聚合市集（fal.ai 已做） |
| 角色包 + 一致性系統 | 通用影片編輯器（CapCut 已做） |
| BYOK 讓用戶自帶 key | 靠轉售 API 賺差價 |
| 中文化 + 垂直場景範本 | 跟 Runway/Pika 打通用功能戰 |

### 護城河

1. **角色一致性系統**（定妝照 + 偶包 + Element Binding）— 市場上沒有現成工具能做到這個穩定度
2. **prompt 自動組裝**（v14 系統）— 用戶不需要懂 prompt 就能出好結果
3. **流程標準化**（4 步驟流程）— 把「試錯」變成「選擇」

### 免費 vs 付費切割（初步共識）

| 層級 | 功能 | 理由 |
|------|------|------|
| 免費 | 基礎引導流程、單一角色包、基本 prompt 生成、BYOK 接入 | 讓人進來體驗 |
| 付費 | 多角色包管理、v14 明星偶包系統、Element Binding 進階控制、批次生成、分鏡範本庫 | 核心壁壘 |
| 高階付費 | API 代管（免自己申請 key）、優先算力、自訂角色包訓練 | 商業用戶 |

### 技術階段

- 第一階段：只支援 Kling（已驗證）
- 第二階段：才加 fal.ai adapter（多 provider）

## 第 2 輪

### 用戶確認目標人群

優先打 **A + B** 兩群人：

**A. 中小企業主 / 電商賣家（含房產）**
- 痛點：沒時間做推廣，光跑客戶就忙不過來
- 實際案例：用戶的房產客戶就是這樣，根本沒時間研究 AI 工具
- 需求：快速出影片，不要學習成本

**B. 內容創作者 / 自媒體**
- 痛點：產量需求大、角色一致性難維持
- 需求：效率工具，每次生成品質穩定

### 市場現況（用戶提供）

**現行影片製作成本（台幣）：**
- 單支影片（請人拍+剪）：$3,000 ~ $5,000
- 包套方案（30-40 支）：$30,000 ~ $40,000
- 還需要聘請小編或剪輯人員
- 痛點：花了錢，品質也不一定滿意

**範本規劃三大類：**

| 類型 | 內容 | 範例 |
|------|------|------|
| 講解/口播 | 虛擬看房、房源介紹、代言人講解 | 15 秒口播介紹商品 |
| MV 類型 | 針對不同受眾的影片版本 | 不同風格的房源展示 |
| 時事/其他 | 時事分享、吃播等主題 | 蹭熱點的短影片 |

**範本獨佔機制：**
- 房產專用範本 → 鎖帳號，避免同業撞內容
- 通用範本 → 開放所有用戶使用

**優先級決策：**
- 先做 A 群（企業主），因為「有賺錢轉換需求」→ 付費痛點最明確
- 核心場景：15 秒口播介紹商品、房源短影片
- B 群（創作者）第二階段再做

### 分析：模型能力 vs 範本需求

三大範本類型的技術可行性：

| 範本類型 | 技術需求 | 目前可行？ | 用什麼模型 |
|---------|---------|-----------|-----------|
| 口播/講解 | AI 角色 + 微動作 + 配音 | ✅ 已驗證 | Kling O3 + Element Binding + 後製配音 |
| MV 類型 | 大動作 + 臉不崩 + 節奏感 | ⚠️ 待測試 | HunyuanCustom 或 Kling O3（需實測） |
| 時事/其他 | 圖片轉影片 + 文字疊加 | ✅ 可做 | Kling O3 Image-to-Video |

**MV 製作流程（目前無「MV 模式」按鈕）：**
- 分段生成（3-8 秒片段）→ 後製軟體對齊 BPM → 組合
- 聲音獨立處理（另一個聲音系統），後製合入

**待測試：HunyuanCustom**
- 在 fal.ai 上有 API
- 512p $0.80/影片、720p $2.40/影片
- 專門做身份一致性，但大動作表現待實測

## 第 3 輪

### 用戶確認測試情境

> 「沈知序穿著白色套裝，在現代風格的樣品屋客廳裡走動，邊走邊微笑看鏡頭，最後轉身指向落地窗外的風景」

### 測試計劃

用同一段情境描述，分別測三個模型，比較臉部一致性 + 大動作表現：

| 測試 | 模型 | 做法 | 預算 |
|------|------|------|------|
| A | HunyuanCustom 512p | 定妝照 + prompt | ~$0.80 |
| B | Kling O3 Standard（不加 Element Binding） | 定妝照 + prompt | ~$1.12 |
| C | Kling O3 Standard + Element Binding | 定妝照 + prompt + elements 鎖臉 | ~$1.12 |

角色素材：
- 定妝照：`videogo/config/characters/shen-zhi-xu/character-final.png`
- 9:16 版本：`videogo/config/characters/shen-zhi-xu/character-final-9x16.png`
- 表情參考：`videogo/config/characters/shen-zhi-xu/references/` 下的 neutral.png, warm.png, serious.png

預估總成本：~$3 美金

### 測試結果（2026-04-03 執行）

實際費用：A $0.80 + B $0.56 + C $0.56 = **$1.92 美金**

| 測試 | 臉部一致性 | 動作執行 | 畫質 | prompt 遵從度 | 結論 |
|------|-----------|---------|------|-------------|------|
| A: HunyuanCustom 512p | 人物太小看不清 | 只有晃動，沒有走向鏡頭/轉身/指窗 | 512p 模糊 | ❌ 極差 | **淘汰** |
| B: Kling O3 Standard | 前期穩定，轉身後看不到臉 | ✅ 走動→微笑→轉身→指窗 | 784p 清晰 | ✅ 好 | 可用 |
| C: Kling O3 + Element Binding | 全程穩定，跟定妝照一致 | ✅ 同 B，動作稍更自然 | 784p 清晰 | ✅ 好 | **最佳** |

**A 淘汰原因：**
1. 費用更貴（$0.80 vs $0.56），720p 要 $2.40 更不划算
2. 構圖錯誤 — 人物在遠景，跟定妝照的中近景完全對不上
3. 場景品質差 — 普通房間，不像「現代樣品屋」
4. prompt 遵從度極差 — 要求的動作（走向鏡頭、轉身、指窗）幾乎沒做到

**B vs C 觀察：**
- 表情幾乎一樣，Element Binding 在臉部一致性上略勝
- 共同問題：換鏡/大動作轉換時有跳接感
- MV 正式製作應分段生成（每段 3-5 秒一個動作）再後製接起來

### 結論

- MV 類型影片確定用 **Kling O3 + Element Binding**（C 方案）
- HunyuanCustom 不列入 VideoGo 支援清單
- 下一步：分段生成測試（拆成 3 段各 5 秒，各一個動作，後製拼接）
