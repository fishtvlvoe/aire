# VideoGo SaaS — 技術架構與部署

## 第 9 輪：CLI 轉 Web App 需要什麼

現在的 videogo 是本機 CLI 工具，要變成 Web App 多了這些東西：
- 用戶帳號系統（登入/註冊）
- 點數系統（扣點/加點）
- 影片儲存（雲端存放 + 下載）
- 排隊系統（多人同時生成）
- 伺服器（部署）

Fish 之前提過 Oracle Cloud，但表示自己沒用過、擔心太複雜。希望過程簡單，只要把 VPS 架起來，Claude 就能直接部署。Fish 已有 Oracle Cloud 帳號。

Fish 還提到一個台灣的平台（Z 開頭），確認是 **Zeabur** — 台灣團隊做的 PaaS 部署平台，一鍵部署，繁體中文介面。

---

## 第 10 輪：Zeabur vs Oracle Cloud 比較

### Zeabur
- 優點：一鍵部署、push 即上線、不用管伺服器、自動重啟/擴展
- 缺點：月費 $15-30、影片存放要另外接 Cloudflare R2 或 S3
- 難度：⭐

### Oracle Cloud
- 優點：免費方案永久免費（2 台 AMD VM + 1 台 ARM VM 最多 4 核 24GB RAM + 200GB 儲存 + 10TB/月流量）
- 缺點：要自己裝環境（SSH、Linux、Node.js、Nginx），掛掉要自己處理
- 難度：⭐⭐⭐⭐

---

## 第 11 輪：混合方案討論

Claude 建議兩種方案：

### 方案 A：Zeabur 主力 + Oracle Cloud 當倉庫
- Zeabur：前端 + API + DB + Worker
- Oracle Cloud（免費）：存影片檔案（200GB）
- 月費約 $15

### 方案 B：全部 Zeabur
- 影片存 Cloudflare R2（前 10GB 免費）
- 少一個平台管理，但費用稍高

Fish 的回應：如果 Claude 能幫忙處理 Oracle Cloud 就用 Oracle，不行就用 Zeabur。

---

## 第 12 輪：確認部署方案

Claude 說明能力邊界：
- 能做：寫部署腳本、設定檔、自動化安裝指令、一步步教學
- 不能做：替 Fish 登入 Oracle 控制台、替 Fish 按按鈕

實際流程：
1. Fish 在 Oracle Cloud 控制台開一台 VM（Claude 一步步教，約 10 分鐘）
2. Fish 把 SSH 連線資訊提供
3. 之後 Claude 全部用指令搞定：裝 Node.js、Nginx、PostgreSQL、部署程式碼、設定自動重啟、設定 HTTPS

**最終決定：Oracle Cloud 主力，Fish 花 10 分鐘開 VM，剩下 Claude 來。**

### 預計架構

```
Oracle Cloud VM（免費）：
├── Next.js 前端（用戶看到的網頁）
├── API 伺服器（處理請求、扣點數）
├── PostgreSQL（用戶帳號、點數、記錄）
├── 背景 Worker（排隊 + 呼叫 Kling API）
└── 影片檔案儲存（200GB 內免費）

外部服務：
├── Kling API（生成影片）
└── 金流 API（收款）
```

費用估算：
- Oracle Cloud：$0（免費方案）
- Kling API：按用量（這是大頭）
- 金流 API：按交易抽成
- 域名 + SSL：幾百台幣/年
