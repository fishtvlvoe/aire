## 2. 模組拆分

### 2.1 前端（Next.js）

| 模組 | 功能 | 介面 |
|------|------|------|
| 用戶認證 | 註冊 / 登入 / 登出 | Web + LINE |
| 影片製作流程 | 7 個決策點的 Step-by-Step 介面 | Web 主力 |
| 模板瀏覽 | 現成模板列表、預覽 Demo 影片 | Web 主力 |
| 角色管理 | 上傳照片 / 建定妝照 / 錄聲音 / 管理「我的角色」 | Web 主力 |
| 點數儀表板 | 點數餘額 / 消耗紀錄 / 購買點數 | Web + LINE |
| 影片庫 | 已生成影片列表 / 下載 / 發布到 PostGo | Web 主力 |
| LINE 輕量視圖 | 收通知 / 快速 Yes/No / 查進度 | LINE Mini App |
| 付款頁面 | 選購點數包 / 月費方案 | Web 主力 |

### 2.2 API Server

| 模組 | 職責 |
|------|------|
| Auth 模組 | 帳號驗證、JWT token 管理 |
| Credits 模組 | 點數加減、餘額查詢、扣點驗證 |
| Video 模組 | 影片生成任務建立、進度查詢、影片記錄管理 |
| Template 模組 | 模板 CRUD、模板使用記錄 |
| Character 模組 | 角色包建立、管理、取用 |
| Payment 模組 | Payment Adapter 呼叫金流 API、付款成功後加點數 |
| Script 模組 | 腳本草稿生成、分鏡拆分 |

### 2.3 Background Worker

| 功能 | 說明 |
|------|------|
| 任務排隊 | 多用戶同時生成影片的排隊管理 |
| Kling API 呼叫 | 依序呼叫 Kling API 生成影片片段 |
| 影片後製拼接 | 各片段拼接成完整影片 |
| 通知發送 | 生成完成後通知用戶（Web / LINE） |
| 扣點 | 生成完成後從用戶帳戶扣除對應點數 |

### 2.4 資料庫（PostgreSQL）

詳見第 4 節 Schema 設計。

## 3. API 設計

以下為主要 Endpoint 清單，依模組分類。

### Auth

| Method | Endpoint | 說明 |
|--------|----------|------|
| POST | `/api/auth/register` | 用戶註冊（送出後自動給 50 免費點數） |
| POST | `/api/auth/login` | 用戶登入，回傳 JWT token |
| POST | `/api/auth/logout` | 登出 |
| GET | `/api/auth/me` | 取得當前用戶資料 |

### Credits（點數）

| Method | Endpoint | 說明 |
|--------|----------|------|
| GET | `/api/credits/balance` | 查詢點數餘額 |
| GET | `/api/credits/history` | 點數消耗 / 加值紀錄 |
| POST | `/api/credits/purchase` | 選購點數包或月費方案（呼叫 Payment Adapter） |

### Videos（影片）

| Method | Endpoint | 說明 |
|--------|----------|------|
| POST | `/api/videos` | 建立影片生成任務（含選角色、腳本、分鏡） |
| GET | `/api/videos` | 列出用戶的影片列表 |
| GET | `/api/videos/:id` | 查詢特定影片及生成進度 |
| GET | `/api/videos/:id/download` | 下載影片 |
| POST | `/api/videos/:id/publish` | 發布到 PostGo（第三階段） |

### Templates（模板）

| Method | Endpoint | 說明 |
|--------|----------|------|
| GET | `/api/templates` | 列出所有可用模板 |
| GET | `/api/templates/:id` | 取得特定模板詳細資訊 |

### Characters（角色）

| Method | Endpoint | 說明 |
|--------|----------|------|
| GET | `/api/characters` | 列出用戶可用的角色（系統角色 + 自建角色） |
| POST | `/api/characters` | 建立自建角色（上傳照片 / 聲音） |
| GET | `/api/characters/:id` | 取得角色詳細資料 |
| DELETE | `/api/characters/:id` | 刪除自建角色 |

### Scripts（腳本）

| Method | Endpoint | 說明 |
|--------|----------|------|
| POST | `/api/scripts/generate` | 根據主題自動產生腳本草稿 |
| POST | `/api/scripts/storyboard` | 將腳本拆分為分鏡 |

### Payment（金流 Adapter）

| Method | Endpoint | 說明 |
|--------|----------|------|
| POST | `/api/payment/checkout` | 建立付款請求（轉給金流 API） |
| POST | `/api/payment/webhook` | 金流 API 回呼（付款成功 → 加點數） |

## 4. 資料庫 Schema 初版

### 4.1 用戶（users）

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | UUID | 主鍵 |
| email | VARCHAR(255) | 唯一，登入用 |
| password_hash | VARCHAR(255) | 加密後密碼 |
| display_name | VARCHAR(100) | 顯示名稱 |
| line_user_id | VARCHAR(100) | LINE 帳號綁定（nullable） |
| plan | ENUM | `free` / `basic` / `pro` |
| created_at | TIMESTAMP | 建立時間 |
| updated_at | TIMESTAMP | 最後更新時間 |

### 4.2 點數（credits）

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | UUID | 主鍵 |
| user_id | UUID | FK → users.id |
| balance | INTEGER | 目前餘額（單位：點） |
| updated_at | TIMESTAMP | 最後更新時間 |

### 4.3 點數異動紀錄（credit_transactions）

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | UUID | 主鍵 |
| user_id | UUID | FK → users.id |
| amount | INTEGER | 正數=加點、負數=扣點 |
| type | ENUM | `register_bonus` / `purchase` / `video_charge` |
| reference_id | UUID | 關聯的影片 ID 或付款 ID（nullable） |
| created_at | TIMESTAMP | 時間戳 |

### 4.4 影片（videos）

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | UUID | 主鍵 |
| user_id | UUID | FK → users.id |
| template_id | UUID | FK → templates.id（nullable，自建流程可無模板） |
| character_id | UUID | FK → characters.id |
| title | VARCHAR(255) | 影片標題 |
| script | TEXT | 確認後的腳本 |
| storyboard | JSONB | 分鏡資料（JSON 陣列） |
| status | ENUM | `draft` / `queued` / `generating` / `done` / `failed` |
| file_path | VARCHAR(500) | Oracle Cloud 上的影片檔案路徑 |
| duration_seconds | INTEGER | 影片長度（秒），用於計算扣點 |
| credits_charged | INTEGER | 實際扣除點數 |
| created_at | TIMESTAMP | 建立時間 |
| completed_at | TIMESTAMP | 生成完成時間（nullable） |

### 4.5 模板（templates）

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | UUID | 主鍵 |
| name | VARCHAR(100) | 模板名稱（如「房仲解說」） |
| type | ENUM | `knowledge` / `product` / `mv`（對應三種影片類型） |
| default_character_id | UUID | FK → characters.id（預設角色） |
| scene | VARCHAR(255) | 場景描述（如「客廳場景」） |
| style | VARCHAR(255) | 風格描述（如「知性風格」） |
| demo_video_url | VARCHAR(500) | Demo 影片 URL（新用戶體驗用） |
| prompt_template | TEXT | 腳本生成的 prompt 範本 |
| is_active | BOOLEAN | 是否啟用 |
| created_at | TIMESTAMP | 建立時間 |

### 4.6 角色（characters）

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | UUID | 主鍵 |
| owner_user_id | UUID | FK → users.id（NULL 表示系統角色，如沈知序） |
| name | VARCHAR(100) | 角色名稱 |
| type | ENUM | `system` / `custom`（系統角色 vs 自建角色） |
| portrait_url | VARCHAR(500) | 定妝照 URL |
| voice_model_id | VARCHAR(255) | Kling create-voice 的聲音模型 ID |
| kling_character_id | VARCHAR(255) | Kling API 的角色識別碼 |
| is_active | BOOLEAN | 是否可用 |
| created_at | TIMESTAMP | 建立時間 |

│                            │                         │
│  ┌─────────────┐   ┌───────▼─────┐                  │

### 6.2 部署方式

| 項目 | 說明 |
|------|------|
| 開 VM | Fish 在 Oracle 控制台操作（約 10 分鐘） |

| 備選 | 若 Oracle Cloud 遇阻 → 改用 Zeabur（$15/月，一鍵部署） |

## 7. 開發 Phase 拆分建議

以下依據「用戶體驗流程」和「最小可驗證功能」排序，所有決策來自討論紀錄。

### Phase 1 — 基礎骨架
**目標：用戶可以登入、有帳號、有點數**

- 帳號系統（註冊 / 登入）
- 點數系統（餘額 / 紀錄 / 扣點邏輯）
- Oracle Cloud VM 部署 + PostgreSQL + Nginx + HTTPS

### Phase 2 — 影片生成核心
**目標：用戶可以用現成模板生成第一支影片**

- 模板系統（現成模板：房仲解說 / 產品介紹 / MV）
- 腳本自動生成（輸入主題 → 產出草稿）
- 分鏡拆分
- Kling API 整合（生成影片 + 拼接）
- 背景 Worker + 排隊系統
- 7 個決策點的 Step-by-Step 介面

### Phase 3 — 付費與商業化
**目標：用戶可以購買點數、系統可以收款**

- Payment Adapter 串接金流 API
- 點數包購買流程
- 月費方案訂閱流程
- 付款成功自動加點數

### Phase 4 — 自建角色（進階功能）
**目標：用戶可以用自己的臉做影片**

- 角色建立流程（上傳照片 → 定妝照 → 錄聲音 → 建立 Kling 角色）
- 「我的角色」管理介面
- 自建角色套用模板

### Phase 5 — LINE Mini App
**目標：手機輕量操作**

- LINE Mini App 輕量介面
- 收通知（影片生成完成）
- 快速 Yes/No 確認
- 查看進度

### Phase 6 — PostGo 整合（第三階段）
**目標：一鍵發布到各平台**

- 串接 PostGo 多平台發布
- 勾選 YouTube / IG Reels / TikTok / FB 等
- 發布狀態追蹤

## 附錄：與 videogo 的關係

現有的 `videogo` 是 VideoGo 的技術原型，本機 CLI 工具。VideoGo SaaS 是將其核心能力（Kling API 影片生成）包裝成多用戶 Web 服務。改名工程（videogo → VideoGo）在 SaaS 架構確定後一起執行。
