# VideoGo SaaS 使用者流程設計

> 日期：2026-04-03
> 狀態：已確認方向，待細化 Spec

---

## 設計原則

1. **不登入可體驗到預覽**，生成才要求登入/付費
2. **一個入口三條路線**，用戶自己選深度，不強制引導
3. **全自動運營**，一人公司不做人工策展
4. **角色兩軌並行**，預設角色庫 + 客戶自訂角色

---

## 首頁三入口

用戶進入首頁看到三張卡片，一眼判斷自己要走哪條：

```
┌─────────────────────────────────┐
│  🔥 熱門主題 — 看看最近什麼正紅  │
│  ✍️ 我有想法 — 貼上你的內容      │
│  ⚡ 快速生成 — 一張圖搞定        │
└─────────────────────────────────┘
```

---

## 路線 A：熱門主題（全引導）

適合：不知道要拍什麼的用戶

```
步驟 1：瀏覽趨勢主題
  ├── 來源：Google Trends / 新聞 API 自動抓取
  ├── 呈現：3-5 個當紅主題卡片
  └── 不分產業，通用趨勢

步驟 2：選切角
  └── AI 針對選定主題自動產出 3-5 個切角建議

步驟 3：選角色
  ├── 從預設角色庫選（免費）
  └── 用自己上傳的角色（付費）

步驟 4：確認/微調腳本
  ├── AI 自動生成完整腳本（台詞 + 表情 + 語速 + 分鏡）
  └── 用戶可微調台詞內容

步驟 5：預覽 → 生成
  ├── 預覽：分鏡圖 + 模擬語音
  └── 生成：需登入，消耗點數
```

---

## 路線 B：自備內容（半引導）

適合：有劇情想法但不會拍的用戶

```
步驟 1：輸入內容
  └── 貼上文字劇情 / 想法（或語音輸入）

步驟 2：回答關鍵問題（3-5 題，不超過 5 題）
  ├── 情緒：開心 / 認真 / 焦慮 / 溫暖
  ├── 語速：快節奏 / 正常 / 慢而穩
  ├── 場景：選現成的 或 上傳自己的
  └── 進階設定（收折）：每句話時長、聲線偏好

步驟 3：選角色
  ├── 從預設角色庫選（免費）
  └── 用自己上傳的角色（付費）

步驟 4：確認腳本 + 分鏡
  ├── AI 根據用戶輸入 + 問答結果，產出腳本 + 分鏡
  └── 用戶確認或微調

步驟 5：預覽 → 生成
  ├── 預覽：分鏡圖 + 模擬語音
  └── 生成：需登入，消耗點數
```

---

## 路線 C：快速生成（零引導）

適合：只想快速出片、不想處理細節的用戶

```
步驟 1：選角色
  ├── 上傳一張照片（系統自動處理）
  └── 或選預設角色

步驟 2：選背景
  └── 從預設背景庫選擇

步驟 3：輸入一段話
  └── prompt 或台詞（一段話搞定）

步驟 4：直接預覽 → 生成
  ├── 系統自動決定：表情、語速、分鏡
  ├── 預覽：免登入
  └── 生成：需登入，消耗點數
```

---

## 角色系統

### 預設角色庫（免費）

- 系統提供 5-10 個已調校好的角色
- 每個角色包含：定妝照、表情參考、Element Binding 設定、預設聲線
- 沈知序為首位角色（已驗證穩定）

### 客戶自訂角色（付費）

```
客戶上傳照片（一張即可）
    ↓
系統自動生成定妝照（AI 修正角度、光線、背景）
    ↓
客戶確認「這張臉對嗎？」
    ↓
系統自動跑 Element Binding 設定
    ↓
完成，可以開始拍片
```

- 聲音：從系統預設聲線庫選擇，第一版不做聲音克隆

---

## 聲音系統

- 第一版：系統預設聲線庫（多種語調風格可選）
- 不做聲音克隆（複雜度高，第二階段再評估）
- 聲音後製合入影片（與影像分開處理）

---

## 趨勢資訊來源

- 全自動：Google Trends API / 新聞 API 自動抓取
- 不做人工策展（一人公司無人力）
- 不分產業，呈現通用趨勢，客戶自行判斷相關性
- AI 自動對每個趨勢產出切角建議

---

## 統一後端流程

三條路線前端引導深度不同，但最終都匯入同一個生成管線：

```
腳本（台詞 + 表情 + 語速）
    ↓
分鏡（每段 3-5 秒，各一個動作）
    ↓
預覽（分鏡圖 + 模擬語音）
    ↓
生成（Kling O3 + Element Binding，分段生成 → 拼接）
    ↓
後製（色調 + 聲音合成 + 字幕）
    ↓
輸出
```

## 技術決策（已確認）

| 項目 | 決策 | 依據 |
|------|------|------|
| 影片生成模型 | Kling O3 Standard + Element Binding | MV 測試結果（09 文件） |
| HunyuanCustom | 淘汰，不列入支援 | 費用貴、畫質差、prompt 遵從度極差 |
| MV 類型製作 | 分段生成（每段 3-5 秒）→ 後製拼接 | 單段換鏡會跳接 |
| 影片格式 | 9:16 直式（短影片優先） | 目標平台：抖音/IG Reels/YouTube Shorts |
