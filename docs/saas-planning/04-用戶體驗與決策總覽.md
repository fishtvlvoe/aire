# VideoGo SaaS — 用戶體驗與決策總覽

> 討論日期：2026-04-03
> 參與者：Fish（老魚）+ Claude Opus

---

## 第 13 輪：新用戶第一次體驗

討論了三種方式：
- A）先建角色再做影片（像 HeyGen）
- B）先選模板再說（像 Canva）
- C）先看 Demo 再決定（試用體驗）

**Fish 選擇：B + C 結合**

流程：
1. 註冊
2. 看 Demo 影片（展示 AI 做出來的效果）
3. 「我也要做一支」→ 選模板
4. 改文字（台詞）→ 預覽 → 生成
5. 第一支影片完成（用掉免費 50 點）
6. 「還想做」→ 付費

設計哲學對應：引擎 4（外化元框架）— 每一步告訴用戶「你在哪、下一步在哪」。

---

## 第 14 輪：模板系統設計

討論了三種模板概念：
- A）模板 = 角色 + 場景 + 風格 已配好（用戶只改台詞）
- B）模板 = 只有風格，角色自己建
- C）兩種都有

**Fish 選擇：C — 兩種都有**

### 現成模板（新手 / 快速體驗）
- 「房仲解說」→ 沈知序 + 客廳場景 + 知性風格
- 「產品介紹」→ 男性角色 + 白底棚拍 + 活潑風格
- 「音樂 MV」→ 女性角色 + 夜景 + 電影感
- 用戶只改台詞就能出片
- 免費 50 點就花在這裡

### 自建角色（進階 / 付費用戶）
- 上傳照片 → 建定妝照 → 錄聲音 → 變成「我的角色」
- 可以套用任何模板風格
- 這是自然的付費轉換點：「我要用自己的臉做片」

## 已確認決策總覽（截至 2026-04-03）

### 1. 產品定位
- AI 內容自動化產線的中間段（Input → VideoGo → PostGo）
- 「不會剪片的人，也能量產專業影片」
- 三種影片：真人知識解說 / 產品介紹 / MV 音樂娛樂
- 改名：videogo → VideoGo（跟 BuyGo、PostGo、PayGo 同系列）

### 2. 自動化程度
- 半自動 — 用戶是導演，系統是劇組
- 7 個決策點全由用戶確認（選主題、選角色、確認腳本、確認分鏡、等待生成、看成品、選平台發布）
- 中間執行過程全自動

### 3. 介面
- Web App（電腦版）= 主力，完整功能（含影片品質確認 4K/8K）
- LINE Mini App = 輕量（看通知 + 快速確認 Yes/No + 看進度）
- 手機不適合做完整影片編輯操作

### 4. 目標用戶
- 房仲
- 自媒體創作者
- 業務
- 中小型企業主
- 製造業與生產業
- 共同特徵：不會寫程式、不會剪片，但需要影片內容

### 5. 商業模式
- 點數制 + 可選月費（兩種並存）
- 1 點 = 1 秒影片
- 成本 NT$1.5/點，售價 NT$3-4/點
- 毛利目標：50-80%（留行銷 + 分潤空間）
- 免費體驗：註冊送 50 點（一次性，不每日送）
- 不能像可靈/即夢那樣大方送點數（他們用自己的模型，我們串 API 有真實成本）

### 6. 新用戶體驗
- 先看 Demo 影片 → 選模板改台詞 → 出片（B + C 結合）
- 模板系統：現成模板（快速體驗）+ 自建角色（進階付費）
- 自建角色是自然的付費轉換點

### 7. 技術架構
- 部署：Oracle Cloud VM（免費方案，Fish 花 10 分鐘開 VM，Claude 用指令部署）
- 前端：Next.js
- 資料庫：PostgreSQL（用戶帳號、點數、記錄）
- 背景 Worker（排隊 + 呼叫 Kling API）
- 影片存放：Oracle Cloud 200GB 免費空間
- 外部服務：Kling API（影片生成）+ 金流 API（收款，Adapter 模式）
- 備選方案：Zeabur（如果 Oracle Cloud 太複雜就改用）

### 8. 設計哲學
- 套用 12 條引擎（來自 BGO 核心框架引擎手冊，同 BuyGo 底層）
- Why 不變，What（具體規則）根據影片領域調整
- 核心精神：不是做一個軟體，是把設計哲學生產線複製到影片領域

### 9. 三段產線全景
- Input（開發中）：另一個 SaaS，從 Dcard/PTT/新聞/Google Trends 抓趨勢主題
- VideoGo（目前在做）：影片產出系統，另有獨立的文字產出和圖片產出系統
- PostGo（第三階段）：多平台自動發布（YouTube/IG Reels/TikTok/FB 等）

### 10. 競品參考
- 定價參考：可靈、即夢、Runway、Domo AI（點數制）
- 功能參考：HeyGen、Synthesia（數位人影片平台）
- 商業模式參考：Adobe/Canva 訂閱制（非 Steam 買斷制）
