# 費用與時程

## 開發分兩個階段

### 第一階段：物件分析助手（現在要做的）

| 項目 | 內容 |
|------|------|
| 功能 | 打一個地址 → 自動分析 → 產出報告 |
| 時間 | 2 個月 |
| 費用 | 10 萬 |
| 付款方式 | 簽約付 5 萬，完成驗收付 5 萬 |

#### 時程細節

| 時間 | 做什麼 | 你需要配合什麼 |
|------|--------|---------------|
| 第 1-2 週 | 做出 demo，用你的真實物件測一次 | 提供一個真實物件資料 |
| 第 3-4 週 | 開發核心功能 | 有問題會找你確認 |
| 第 5-6 週 | 補齊所有功能 | 有問題會找你確認 |
| 第 7-8 週 | 你實際使用測試，我修 bug | 用真實物件測 3-5 個 |

### 第二階段：行銷自動化（之後再做）

| 項目 | 內容 |
|------|------|
| 功能 | 行銷文案生成、一鍵上架準備、影片劇本 |
| 時間 | 另議 |
| 費用 | 另議（預估 6-10 萬） |
| 前提 | 第一階段做完、你覺得好用，才做第二階段 |

兩個階段是**同一個網站**，第二階段做完後在同一個系統裡多出行銷功能。

## 上線之後的費用

| 項目 | 每月費用 | 說明 |
|------|---------|------|
| AI 分析費 | 500-1,500 | 你用多少付多少 |
| 雲端主機 | 500-2,000 | 系統放在網路上的費用 |
| 維護支援 | 2,000-3,000 | 修小問題、保持系統正常 |
| **合計** | **3,000-6,500/月** | |

## 合約保障

| 保障項目 | 內容 |
|----------|------|
| 做的範圍 | 寫在合約附件（PRD），白紙黑字 |
| 修改次數 | 含 3 次介面調整 + 2 次功能調整 |
| 超出範圍 | 需要另外討論費用 |
| 保固期 | 上線後 30 天，修 bug 不收費 |
| 程式碼 | 付清全款後給你 |
| 如果不滿意 | 簽約只付 5 萬，看到 demo 再決定要不要繼續 |

## 你的風險

| 風險 | 怎麼保護你 |
|------|-----------|
| 花了錢但不好用 | 先付 5 萬看 demo，不好用可以停 |
| 做出來跟想的不一樣 | 有合約寫死範圍，做什麼不做什麼很清楚 |
| 上線後系統壞掉 | 30 天保固 + 每月維護費 |
