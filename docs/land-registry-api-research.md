# 地政資料 API 串接調查

> 調查日期：2026-05-12
> 目的：AIRE 不動產說明書系統要接「乾淨的源頭資料」，取代現有 OCR 流程

## 背景

永慶不動產的「i智慧系統」能直接介接地政機關拉謄本資料，AIRE 要達到同等級的資料品質。
核心需求：輸入地址/地號 → 系統自動拉謄本 → 結構化資料帶入說明書欄位。

---

## 三個資料管道

### 管道一：地政整合資訊服務共享協作平台（推薦，企業級 API）

| 項目 | 內容 |
|------|------|
| 網址 | https://cop.land.moi.gov.tw/ |
| 主管機關 | 內政部地政司 |
| 民間企業可用 | 2019 年起開放民間企業付費介接 |
| 已有企業用戶 | 永慶不動產、國泰建設、僑馥建築等 15 家 |
| API 數量 | 35 種 API + 5 種 WMS（地圖）+ 5 種 WFS（圖徵）|
| 計費方式 | 以「筆」計費，按用量付費 |
| 累計資料流量 | 17 億筆 |
| 申請方式 | 到平台註冊帳號 → 申請介接 → 審核通過後串接 |
| 限制 | 平台頁面需登入才能看 API 清單和技術文件 |

**這是永慶在用的同一套系統。** AIRE 走這條路就能拿到同等級的資料源。

#### 預估可用的 API（需登入確認）

- 登記謄本查詢（產權、面積、權利範圍、他項權利）
- 地價查詢（公告地價、公告現值）
- 門牌轉地建號（地址 → 地號/建號）
- 地籍圖查詢
- 建物測量成果圖
- 異動索引（交易歷史）
- 土地使用分區

### 管道二：全國地政電子謄本系統（個人/零售，非 API）

| 項目 | 內容 |
|------|------|
| 網址 | https://ep.land.nat.gov.tw/ |
| 性質 | 網頁操作，一筆一筆查，**無 API** |
| 費用 | 每張謄本 20 元，門牌查詢 10 元 |
| 繳費 | 併入中華電信月帳單（需 HiNet 帳號）|
| 可查項目 | 登記謄本、地價謄本、地籍圖、建物測量成果圖、異動索引、異動清冊、門牌查地建號 |
| 服務時間 | 週一～六 08:00-21:00 |
| 法律效力 | 等同臨櫃取得的紙本謄本 |

**不適合系統串接**，但可作為 fallback（業務手動查一筆下載 PDF）。

### 管道三：Twinkle Hub（政府開放資料，免費）

| 項目 | 內容 |
|------|------|
| 網址 | https://hub.twinkleai.tw/ |
| 性質 | MCP 服務，整合 data.gov.tw 52,960 個資料集 |
| 費用 | Alpha 階段免費 |
| 可查 | 實價登錄、都市計畫、使用分區、建築執照、違規名單 |
| **不可查** | **謄本、地籍、產權登記、他項權利** |
| 技術 | MCP 協議，endpoint: https://api.twinkleai.tw/mcp/ |

**補充資料源**，適合填入說明書的行情和環境章節，但不是核心。

## AIRE 資料源分層策略

```
第一層（Twinkle Hub — 免費，現在就能接）
  • 實價登錄行情 → 說明書第 3、4 章
  • 使用分區 → 說明書第 2 章
  • 周邊設施 / 嫌惡設施 → 說明書第 7 章

第二層（地政協作平台 API — 付費，需申請 1-3 個月）
  • 電子謄本 → 說明書骨幹（產權、面積、權利範圍）
  • 地價資料 → 說明書第 5、6 章
  • 地籍圖 / 建物測量成果圖 → 附件

第三層（業務手動補 — 系統留入口）
  • 現場照片、漏水/增建狀況、鄰居訪查
  • 只有人到現場才知道的資訊
```

## 永慶 i智慧系統參考（競品分析）

### 系統定義

永慶房產集團歷經十多年迭代的核心作業平台，橫跨買方、賣方、經紀人三端。
從「i智慧經紀人」演進到「i+智慧」再結合 AI 的「i特助」。

### 與 AIRE 的定位差異

| | AIRE | 永慶 i智慧 |
|--|------|-----------|
| 定位 | 房仲業務的「文件 AI 助手」| 房產集團的「全方位交易平台」|
| 核心價值 | 自動化產出不動產說明書 | 媒合買賣方 + 行銷 + 管理 |
| 用戶 | 個別房仲業務 | 買方 + 賣方 + 經紀人（三端）|
| 規模 | 輕量工具（SQLite、本機部署）| 企業級雲端（數百人研發）|

### AIRE 可參考的設計思路

1. **七審檢核機制** — 每一關都有 AI 或規則把關，沒過不能進下一步
2. **實價登錄整合** — 自動填入周邊行情（內政部有公開 API + Twinkle Hub）
3. **防呆流程** — 必填欄位檢核才能往下走
4. **即時回報** — 文件完成後通知業務/屋主

### 永慶有但 AIRE 不需要追的功能

- AI 物件配對（30 年數據 + ML 推薦）
- AI 煥裝/清空（GenAI 模擬裝潢）
- LINE i特助（24hr 推播找房）
- 全台聯賣系統

### 三層責任歸屬

| 來源 | 標記方式 | 責任歸屬 |
|------|---------|---------|
| 地政 API 拉的 | 系統自動帶入，標記「來源：地政」| 政府資料，系統只是搬運 |

## 不動產說明書法規要求（不動產經紀業管理條例）

### 強制簽章流程

經紀人簽章 → 委託人（屋主）簽章確認 → 向買方詳實解說 → 買方簽收簽章

### 應記載事項

- 產權（權利範圍、抵押權）
- 使用管制（土地使用分區）
- 現況（凶宅、海砂屋、漏水、增建）
- 半徑 300m 嫌惡設施或重要設施

### 不得記載事項

- 禁止「本說明書僅供參考」
- 禁止預測房價上漲
- 禁止模糊面積名詞（必須以謄本為準）

### 罰則

記載不實或未依法解說：6~30 萬罰鍰，經紀人可能申誡或撤銷證照
