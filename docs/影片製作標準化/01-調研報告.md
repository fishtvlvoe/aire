# 影片數位人角色一致性 — 調研報告

> 調研日期：2026-04-02
> 目的：找出維持角色一致性（臉部、聲音、色調、表情）的業界標準做法，避免每次生成結果不同

---

## 一、現有問題

1. **臉部變形**：只靠 prompt 文字描述控制，Kling 每次生成的臉部特徵不完全一致
2. **表情過度**：咀嚼肌用力、嘴巴張太大，不符合角色「平靜自信」的個性
3. **色調不一致**：用 FFmpeg 後製調色，不是 Kling 直出
4. **尺寸錯誤**：沒帶 aspect_ratio 參數，輸出 2:3 需裁切
5. **每次結果不同**：prompt 沒有固定模板，每次人工組裝

## 二、Kling 3.0 官方功能（我們之前沒用到的）

### Element Binding（臉部鎖定）

- 上傳 3-7 張角色不同角度的照片 → Kling 建立「元素庫」→ 取得 element_id
- 之後生成影片時帶入 element_id → 臉部特徵被鎖定，不會變形
- 即使角色被遮擋或轉頭，臉部仍然穩定
- fal.ai API 支援 `element` 參數

**來源**：[Solving Character Inconsistency (Atlas Cloud)](https://atlascloud.ai/blog/solving-character-inconsistency-a-guide-to-kling-3.0-image-to-video-mode)

### SCALE Prompt 框架（官方推薦）

| 層 | 內容 | 作用 |
|---|------|------|
| **S**hot | 鏡位 + 運鏡（medium shot, static camera） | 畫面穩定 |
| **C**haracter | 角色 + 外貌 + `[Character: 描述, Voice Tone]` 標籤 | 角色一致 |
| **A**ction | 動作描述 + 時間軸 | 動作自然 |
| **L**ighting & Location | 燈光 + 場景 + **色調** | 氛圍一致 |
| **E**xtra | 音頻 / 風格 / 技術規格（aspect_ratio, duration） | 技術參數 |

**來源**：[Kling 3.0 Prompting Guide (fal.ai)](https://blog.fal.ai/kling-3-0-prompting-guide/)

### 角色標籤格式

```
[Character: 沈知序, calm confident soft voice, relaxed jaw]: "台詞"
```

明確標籤比自由描述更精確，Kling 會將聲音、嘴型、表情與角色標籤綁定。

**來源**：[Kling 3.0 Prompt Guide (Atlabs)](https://www.atlabs.ai/blog/kling-3-0-prompting-guide-master-ai-video-generation)

### API 參數

| 參數 | 作用 | 我們之前有用嗎？ |
|------|------|---------------|
| `aspect_ratio: "9:16"` | 直接輸出 9:16 | ❌ 沒用，導致要裁切 |
| `element_id` | 臉部鎖定 | ❌ 沒用 |
| `voice_id` | 聲音鎖定 | ✅ 有用 |
| `generate_audio: true` | AI 語音 + 嘴型 | ✅ 有用 |

**來源**：[Kling O3 API (fal.ai)](https://fal.ai/models/fal-ai/kling-video/o3/standard/image-to-video/api)

## 四、可能的失敗點

| 失敗點 | 原因 | 解法 | 成本 |
|--------|------|------|------|
| Element Binding API 不支援 O3 | 文件說 v3 支援，O3 需驗證 | 先跑一次 API 測試 | ~NT$35 |
| 色調在 prompt 控制不精確 | Kling 可能忽略色調描述 | 不行再回 FFmpeg 後製 | 0 |
| aspect_ratio 參數不生效 | API 文件說支援，需驗證 | 同上一次 API 測試 | 0（同次） |
| 表情仍然過度 | prompt 控制有極限 | Element Binding 比純 prompt 更強 | 0（同次） |

## 五、行動計畫

1. 更新程式碼（character-profile YAML + SCALE prompt + API 參數）— 不花錢
2. 一次 API call 驗證 4 件事（element_id + aspect_ratio + 色調 + 表情）— 花 ~NT$35
3. 根據結果微調 prompt — 最多再花 1-2 次 ~NT$70
4. 確認後固定為標準模板 — 之後每支影片都自動用這套
