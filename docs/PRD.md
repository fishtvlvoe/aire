# PRD：不動產物件分析助手

## Problem Statement

房仲業務員手頭物件多但沒時間逐一分析。每接到一個新物件，要手動查實價登錄、上 591 比對同區開價、研究周邊設施、判斷開價合理性、準備議價策略 — 這個過程每個物件要花 1-2 小時。導致業務員只能挑少數物件深入分析，其餘憑經驗快速判斷，增加了簽到「爛單」（開價不合理、物件有風險）的機率。

## Solution

建立一個 RWD Web App（手機瀏覽器可開），房仲只要輸入一個地址，系統在 3 分鐘內自動產出完整的物件分析報告。報告包含實價登錄數據分析、周邊設施、建議售價區間、議價策略、SWOT 分析和屋主經營指南。所有數字來自確定性計算（非 AI 猜測），每個數字可追溯到原始資料來源。

系統同時自動生成 591、好房網、永慶、信義等平台的搜尋連結，房仲點一下就能看到同區目前在售物件，截圖傳回來後系統會更新分析。

## User Stories

1. 作為房仲，我要能輸入一個地址就取得完整分析報告，以便在 3 分鐘內判斷這個物件值不值得接
2. 作為房仲，我要看到附近 3 公里內近 1 年的實價登錄成交紀錄，以便知道這個區域的真實成交行情
3. 作為房仲，我要看到成交均價、中位數和每坪單價，以便有數據依據跟屋主談價格
4. 作為房仲，我要看到價格趨勢（上漲或下跌），以便判斷現在是不是好的銷售時機
5. 作為房仲，我要能點開每一個數字看到原始成交紀錄，以便驗證系統給的數字是否正確
6. 作為房仲，我要看到資料的信心度標示，以便知道樣本數夠不夠、分析可不可靠
7. 作為房仲，我要看到資料的時間範圍，以便知道這些數字是多久以前的
8. 作為房仲，我要看到周邊最近的學校、醫院、公園、車站和超市的名稱和距離，以便跟買方介紹生活機能
9. 作為房仲，我要看到生活機能評分（1-10 分），以便快速判斷這個地段的吸引力
10. 作為房仲，我要看到建議售價區間，以便跟屋主談合理的開價
11. 作為房仲，我要看到屋主開價高於行情多少百分比，以便有具體數字說服屋主降價
12. 作為房仲，我要看到以目前開價預估要賣多久，以便告訴屋主「堅持這個價可能要賣一年」
13. 作為房仲，我要看到 3 種不同的議價話術，以便根據屋主的個性選擇合適的談法
14. 作為房仲，我要看到買方心理分析，以便知道這個價位的買家通常在意什麼
15. 作為房仲，我要看到物件的優勢和劣勢，以便知道怎麼包裝賣點、怎麼處理缺點
16. 作為房仲，我要看到屋主經營指南（溝通策略、回報頻率、降價時機），以便簽委託後有效經營屋主關係
17. 作為房仲，我要能一鍵取得 591、好房網、永慶、信義、Google Maps 的同區搜尋連結，以便快速瀏覽同區在售物件
18. 作為房仲，我要能把 591 等平台的截圖傳回系統，以便讓 AI 比較「目前開價」和「實際成交價」的落差
19. 作為房仲，我要能輸入物件的抵押/假扣押狀況，以便系統標示風險等級
20. 作為房仲，我要能輸入自己的經驗判斷（例如「屋主急售」），以便 AI 將這些資訊納入分析
21. 作為房仲，我要能看到所有分析過的物件列表，以便管理手頭上的所有案子
22. 作為房仲，我要能標記物件狀態（未分析/已分析/已簽委託/已成交/放棄），以便追蹤每個案子的進度
23. 作為房仲，我要能搜尋和篩選物件（按區域、價格、狀態），以便快速找到特定案子
24. 作為房仲，我要能查看任何物件的歷史分析報告，以便回顧之前的分析結果
25. 作為房仲，我要能在手機瀏覽器上流暢操作系統，以便在外出帶看時也能使用
26. 作為房仲，我要看到免責聲明標示，以便確認這不是正式估價報告
27. 作為房仲，當自動查詢失敗時我要能手動輸入物件資料，以便不被技術問題阻擋
28. 作為房仲，我要能拍照上傳謄本讓 AI 辨識，以便在沒有地址時也能使用系統

## Implementation Decisions

### 系統架構

- **前端**：Next.js，RWD 設計，類 ChatGPT 串流式介面
- **後端**：Next.js API Routes
- **資料庫**：PostgreSQL（透過 Supabase 託管）
- **AI 引擎**：Claude API
- **部署**：Vercel（serverless）

### 外部 API 串接

| API | 用途 | 費用 |
|-----|------|------|
| Google Geocoding API | 地址→經緯度座標 | 免費 10,000 次/月 |
| Google Places Nearby Search | 查周邊設施 | 免費 10,000 次/月 |
| 國土測繪中心 API | 座標→行政區/地段 | 免費 |

### 資料管線

- 實價登錄 Open Data CSV：每月 1/11/21 日排程從 `plvr.land.moi.gov.tw` 下載
- CSV 解析後匯入 PostgreSQL，按縣市、行政區、地段建索引
- 查詢時用經緯度座標 + 半徑 3 公里做地理範圍查詢

### AI 分析引擎設計原則

- **LLM 是翻譯官，不是分析師**：所有數字由確定性計算（SQL/程式碼）產出，LLM 只負責將結構化數據翻譯成自然語言
- **三層防護**：Input Guardrails → RAG + Grounding → Output Validation
- **來源追溯**：每個輸出數字綁定 source_record_ids，前端可展開查看
- **信心度分級**：≥20 筆高信心 / 10-19 筆有限 / 5-9 筆警告 / <5 筆不給估價
- **降級機制**：驗證失敗時不顯示數字，改為「資料不足」

### 搜尋連結生成

系統根據物件所在行政區，自動組合以下平台的搜尋 URL：

| 平台 | URL 格式 | 難度 |
|------|---------|------|
| 好房網 | 路徑式中文（`/region/臺南市_東區/`） | 低 |
| 永慶房屋 | 路徑式中文（`/region/臺南市-東區_house/`） | 低 |
| 591 售屋 | 查詢參數（`?regionid=15&section=67`） | 低（需代碼表） |
| 信義房屋 | 英文路徑（`/Tainan-city/East-district`） | 中（需代碼表） |
| Google Maps | 標準格式（`?query=台南市東區+房屋`） | 低 |

### 報告呈現

- 類 ChatGPT 串流式介面，一段一段產出
- 卡片式區塊（基本資訊、價格分析、周邊設施、議價策略各一塊）
- 每個數字旁有「📊」圖示，點擊展開來源資料
- 報告底部顯示免責聲明和資料時間戳

### 物件管理

- CRUD 操作：新增、查看、更新狀態、刪除
- 狀態流：未分析 → 已分析 → 已簽委託 → 已成交 / 放棄
- 列表支援按區域、價格區間、狀態篩選

### 搜尋範圍

- 實價登錄查詢：以物件座標為中心，半徑 3 公里
- 周邊設施查詢：以物件座標為中心，半徑 3 公里
- 平台搜尋連結：同行政區

## Testing Decisions

### 測試原則

- 只測試外部行為（輸入什麼 → 輸出什麼），不測試內部實作細節
- 重點測試涉及金錢數字的計算邏輯（均價、中位數、單價）
- AI 生成的文字內容不做精確比對，只驗證結構和必要欄位存在

### 需要測試的模組

**1. 實價登錄計算引擎**（最重要）
- 給定一組成交紀錄，驗證均價計算正確
- 給定一組成交紀錄，驗證中位數計算正確
- 給定一組成交紀錄，驗證每坪單價範圍正確
- 給定不同時間的成交紀錄，驗證趨勢計算正確
- 給定 0 筆、3 筆、10 筆、30 筆紀錄，驗證信心度分級正確
- 給定座標和半徑，驗證地理範圍查詢只回傳範圍內的紀錄

**2. 搜尋連結生成器**
- 給定行政區名稱，驗證 591 URL 正確
- 給定行政區名稱，驗證好房網 URL 正確
- 給定行政區名稱，驗證永慶 URL 正確
- 驗證所有生成的連結格式合法

**3. Geocoding 模組**
- 給定有效地址，驗證回傳座標在合理範圍
- 給定無效地址，驗證回傳錯誤而非錯誤座標

**4. AI 輸出驗證層**
- 驗證 AI 輸出的數字與計算結果一致
- 驗證資料不足時系統降級（不給估價）

## Out of Scope

以下功能明確不在本次 PRD 範圍內：

### 產品線 B（行銷自動化，未來另議）
- 行銷文案生成（3 種風格）
- 一鍵上架準備（591/房屋網）
- 影片劇本 / 分鏡腳本
- AI 影片生成提示詞
- 行銷歌曲 + MV 腳本

### 未來規劃
- 即時對話錄音分析（與屋主談話時的即時判斷）
- 多使用者帳號管理（SaaS 化時再做）

### 明確不做
- 手機原生 APP（RWD 網頁足夠）
- 自動爬蟲 591 / 永慶 / 信義（法律風險）
- 正式不動產估價報告（需估價師資格）
- 凶宅 / 違建 / 海砂屋查詢（無可靠資料源）

## Further Notes

### 風險管理

本 PRD 經過熵減倒推分析（詳見 `docs/discovery/14-失敗清單-熵減倒推分析.md`），共識別 24 個失敗可能性，排除率 88%。主要殘留風險為黑天鵝事件（房市崩盤、法規變動、大公司搶市場），已有應對方案但無法完全排除。

### 法律注意事項

- 所有報告必須包含免責聲明：「本分析為市場參考資訊，非正式不動產估價報告」
- 不使用「估價」二字，改用「市場參考分析」
- 不自動爬取任何第三方平台資料

### 資料來源與更新

| 資料 | 來源 | 更新頻率 | 延遲 |
|------|------|---------|------|
| 實價登錄成交 | 內政部 Open Data | 每月 3 次 | 1-3 個月 |
| 周邊設施 | Google Maps API | 即時 | 無 |
| 行政區/地段 | 國土測繪中心 | 即時 | 無 |
| 同區開價 | 客戶手動提供 | 即時 | 無 |
